开始前 3 项检查
  1. USB-C 线必须是数据线（纯充电线识别不到串口）
  2. 过程约 30 分钟, 期间 Mac 不能睡眠 — 命令都带了 caffeinate -i 前缀防睡眠
  3. 第 1 步的备份是唯一完整后悔药, 没备份别刷

0准备: 装 esptool + 找串口

pip3 install esptool

# 插上 StackChan 后:
ls /dev/cu.usbmodem*
# 输出类似 /dev/cu.usbmodem14201 —— 记下, 后面命令替换 <串口>
没输出? 换数据线/换 USB 口。CoreS3 是原生 USB, macOS 无需驱动。仍不行: 按住机身 BOOT 键插电(进下载模式)再 ls。

1全片备份（必做, ~4 分钟）

cd ~/Desktop
caffeinate -i python3 -m esptool --chip esp32s3 --port <串口> --baud 921600 \
  read_flash 0x0 0x1000000 stackchan-factory-backup.bin

# 校验大小, 必须恰好 16777216:
stat -f%z stackchan-factory-backup.bin
备份传一份到 EC2 存档（可选但推荐）:
scp -i <你的密钥.pem> stackchan-factory-backup.bin ec2-user@18.217.255.9:~/worklog/stackchan/

2取固件并校验

git clone https://github.com/CrypticDriver/stackchan-embodied
cd stackchan-embodied/firmware/build-artifacts
shasum -a 256 -c SHA256SUMS      # 5 个 OK 才继续
已 clone 过就 git pull —— 固件在 2026-07-11 有连接性修复, 必须用最新版 (stack-chan.bin 应为 e92179fa 开头)。

3刷入（~2 分钟）

caffeinate -i python3 -m esptool --chip esp32s3 --port <串口> --baud 460800 \
  write_flash --flash_mode dio --flash_size 16MB --flash_freq 80m \
  0x0 bootloader.bin \
  0x8000 partition-table.bin \
  0xd000 ota_data_initial.bin \
  0x20000 stack-chan.bin \
  0xba0000 generated_assets.bin
报 Failed to connect: 按住 BOOT 键重新插电再跑。写入不稳: --baud 降到 115200。

4配网

  • 刷完自动重启。若原 WiFi 失效, 设备开热点 → 手机连上 → 配网页选家里 2.4G WiFi
  • ⚠️ 若配网页有「高级选项/OTA 地址」, 确认或填入:
    https://da8daz4hvc7q8.cloudfront.net/xiaozhi/ota/
    (编译期已内置此地址; 但若 NVS 残留旧小智地址, 手动改一次即覆盖)
